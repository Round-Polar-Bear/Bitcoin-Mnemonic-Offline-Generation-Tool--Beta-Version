## 执行摘要

v4.0 对单文件离线助记词生成工具进行了结构精简和多层安全加固。核心目标是确保生成的 BIP39 助记词能够稳定恢复对应的 BIP84 首地址，并在任何校验失败时停止生成、清除结果。

本版本固定生成：

- Bitcoin 主网地址
- Native SegWit P2WPKH 地址，格式为 `bc1q...`
- 派生路径 `m/84'/0'/0'/0/0`
- BIP39 Passphrase 为空

## 关键安全更新

### 1. 强制使用原生 WebCrypto

- 删除自定义 SHA-256、SHA-512、HMAC-SHA512 和 PBKDF2 降级实现。
- 强制要求浏览器支持 `crypto.getRandomValues` 和 `crypto.subtle`。
- 初始化时使用标准测试向量验证 SHA-256、HMAC-SHA512 和 PBKDF2-SHA512。
- 任一能力缺失或测试失败时，生成按钮保持禁用。

### 2. 增加生成结果自证

- 生成助记词后，重新解码并验证 BIP39 checksum。
- 确认反向解码能够完整还原原始熵。
- 地址执行两次完整 BIP84 派生，两次结果一致后才显示。
- 显示地址前验证 Bech32 校验和、主网 HRP、Witness Version 和 Witness Program 长度。

### 3. 修复生成任务竞态

- 生成期间锁定生成、清除、复制和词数切换按钮。
- 新生成开始时立即清空旧地址。
- 使用任务编号阻止过期异步任务覆盖当前结果。
- 地址校验失败时清除本次助记词，采用 fail-closed 策略。

### 4. 强化敏感数据清理

- 熵在成功和异常路径中均通过 `finally` 清零。
- 种子和最终私钥使用后立即清零。
- BIP84 主密钥、中间私钥和 chain code 在异常路径中也会清零。

### 5. 缩小攻击与审计面

- 删除浏览器扩展注入的样式内容。
- 删除未使用的 Base58、未压缩公钥和旧测试代码。
- 移除正式页面暴露的全局测试函数。
- 保持严格 CSP，禁止网络连接和外部资源。

## 验证结果

本版本经过以下独立验证：

- 600 组随机 12/24 词助记词反向还原，差异为 `0`
- 600 组独立 BIP84 地址交叉验证，差异为 `0`
- WebCrypto 缺失模拟能够正确 fail-closed
- 构造的错误主网、测试网、Taproot 和无效 Bech32 地址均被拒绝
- 浏览器实际生成流程、控件锁定和地址展示正常
- 静态扫描未发现网络请求、外部脚本、动态代码执行或 `Math.random`

## 兼容性说明

本版本必须运行在支持完整 WebCrypto API 的现代浏览器中。部分浏览器可能限制 `file://` 页面的 WebCrypto 能力；遇到安全自检失败时，请在完全断网的可信设备上通过本地服务器打开页面。

```bash
python3 -m http.server 8000 --bind 127.0.0.1
```

然后访问：

```text
http://127.0.0.1:8000/Bitcoin-Mnemonic-Offline-Generation-Tool-SECURE.html
```

## 使用要求

- 在完全断网、禁用浏览器扩展的干净环境中运行。
- 将助记词手写保存，不要截图、复制到云端或发送给任何人。
- 使用另一款可信离线钱包恢复助记词，并核对路径 `m/84'/0'/0'/0/0`。
- 确认恢复出的首地址与页面地址完全一致。
- 首次使用先进行小额收款和发送测试。
