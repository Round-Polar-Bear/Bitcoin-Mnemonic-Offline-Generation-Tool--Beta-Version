# 变更日志 v5.0–v5.3

## v5.3 — 2026-07-05

安全修复（自检失败后的"生成按钮复活"漏洞）：

- 修复：`initEmbeddedWordlist` 先把词表写入 `WORDLIST` 再做哈希校验——若内置词表被篡改，哈希校验抛错后 `WORDLIST` 仍持有 2048 个被篡改的单词，可通过生成入口的 `length===2048` 守卫。现改为先校验哈希、通过后才写入。
- 修复：`updatePassphraseValidation()` 无条件重算 `btnGenerate.disabled`——安全自检失败后，只要在 Passphrase 输入框打字或开关面板，生成按钮就会被重新启用。两个缺陷叠加时，被篡改的词表可以在无任何报错的情况下用于生成助记词（反向校验和用同一份篡改词表、地址自检向量不依赖词表，均无法拦截）。
- 加固：新增 `securityChecksPassed` 门闩，`updatePassphraseValidation`、`setGenerationControlsLocked` 和生成按钮点击入口三处全部叠加该门闩；自检未通过时任何 UI 路径都无法启用或触发生成。
- 回归防护：`verify-security-invariants.js` 新增 5 条不变量（门闩存在、三处引用、哈希先于写入的顺序检查），共 30 条；顺序检查已验证对新旧两种代码顺序分别通过/报错。
- 已用独立测试模拟篡改词表场景：篡改词表被拒绝且 `WORDLIST` 保持为空；正版词表与全部 BIP39/BIP84/RIPEMD160 回归向量通过。

## v5.2 — 2026-07-03

体验与纵深防御加固（本轮未修改任何密码学核心逻辑，BIP39/BIP32/BIP84/RIPEMD160/Bech32 均已用独立测试重新验证）：

- 修复：`.passphrase-validation` 引用了未定义的 CSS 变量 `--danger`，导致"两次口令不一致"提示不显示警示色；已补充 `--danger`/`--danger-bg` 变量（含深色模式）。
- 修复：切换 12/24 词按钮不会作废已生成的助记词和地址，导致按钮状态与屏幕内容不一致；现在切换词数会立即清空旧结果，与 Passphrase 面板的既有行为保持一致。
- 新增：助记词逐词编号显示，降低手写抄录时漏词、错序的概率。
- 优化：修复序号元素误继承单词卡片样式的问题；序号改为固定宽度右对齐、单词统一左对齐，并修正小屏双列宽度计算，确保 1–24 和长单词整齐排列。
- 新增：生成成功后可选的"备份核对"——随机抽取两个位置，要求手动回填对照，帮助在离开页面前发现抄写错误；核对框内容会被"清除"按钮和页面卸载清理一并清空。
- 新增：可选的"补充熵（高级）"输入——填写内容会与 `crypto.getRandomValues` 输出逐字节异或后再用于生成助记词，用作纵深防御；留空或填弱内容都不会降低安全性，也绝不能替代安全随机数单独使用。
- 新增：备份核对的位置抽取使用基于 `crypto.getRandomValues` 的无偏拒绝采样实现（`secureRandomInt`），不引入 `Math.random`。
- 代码清理：简化 `pointAdd` 中 x1==x2 且 y1!=y2 分支的冗余判断；为 `scalarMult` 非恒定时间实现补充说明其威胁模型下可接受的原因。
- 无障碍：补充熵输入框补上 `aria-describedby` 关联提示文案；备份核对的两个输入框补上 `aria-describedby` 关联提示与结果区域。
- 交互：备份核对的两个输入框支持回车直接触发核对，无需额外点击“核对”按钮。
- 修复：`#btn-verify-backup` 沿用了全局按钮的 `margin: 0.5rem 0`，与 `.backup-check-container` 的 `align-items: flex-end` 叠加后导致“核对”按钮比两个输入框高出一截；已清零该按钮的 margin，并在小屏下与输入框一起改为整行宽度。

## v5.1 — 2026-07-03

安全与可靠性加固：

- Passphrase 改为双输入确认，不一致时禁止生成。
- 任何 Passphrase 输入变化都会立即清除旧助记词与旧地址。
- 关闭 Passphrase 面板或点击“清除”会同时清空两个口令框和派生结果。
- 增加弱口令风险提示，不再称为“第 13/25 个词”。
- 页面内置自检增加 `TREZOR`、中文/Emoji 和 Unicode Passphrase 向量。
- 独立测试扩充为 8 组 BIP84 向量，并接入 `npm test`。
- 增加 PBKDF2、BIP32 HMAC 输入及中间缓冲区的尽力清零。
- 增加打印保护，降低助记词进入打印预览或系统打印缓存的风险。
- 修正文档：唯一推荐 Passphrase 版本；删除 `npx serve`；本地服务器固定绑定 `127.0.0.1`。
- 明确页面内词表哈希不能替代文件外部签名或可信发布哈希。
- 开发依赖移动到 `devDependencies`，依赖审计为 0 个已知漏洞。
- 增加 `.gitignore`，发布包不再携带 `node_modules`。

## v5.0 — 2026-07-02

- 新增可选 BIP39 Passphrase。
- 使用 BIP39 规定的 NFKD 归一化及 PBKDF2-HMAC-SHA512。
- 固定派生 Bitcoin 主网 BIP84 P2WPKH 首地址：`m/84'/0'/0'/0/0`。
- 页面显示 Passphrase 是否启用，但不显示原文。

## 使用要求

- 必须在完全断网、禁用扩展和密码管理器的可信设备上运行。
- 助记词与 Passphrase 必须分开、准确、离线备份。
- 使用另一款可信离线钱包核对路径和首地址。
- 正式使用前完成小额收款与恢复发送测试。
- 页面清理只是尽力减少残留，不能保证擦除浏览器内存或系统交换区。
