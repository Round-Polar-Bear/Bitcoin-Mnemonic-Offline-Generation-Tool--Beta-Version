# 哈希校验操作指南（小白版）

> 适用版本：**v5.3**（2026-07-05）
> 适用文件：`Bitcoin-Mnemonic-Offline-Generation-Tool-SECURE-Passphrase.html`

---

## 这是干嘛的（一句话）

哈希就像文件的"指纹"——文件内容哪怕改动一个字，指纹就完全变样。
你在联网电脑上记下指纹，把文件拷到离线设备后再算一次，**两边一样 = 文件没被动过，可以放心用**。

---

## 当前版本的指纹（v5.3）

```
dc80cd3ee40ae0e06a5ff5eab08daf849e837a72625f75ea991d9b14e5bd80ec
```

速记（至少核对这两段）：

- 开头 8 位：`dc80cd3e`
- 结尾 8 位：`e5bd80ec`

---

## 第一步：把指纹抄在纸上

用笔把上面的指纹抄在纸上（至少抄开头 8 位 + 结尾 8 位）。

⚠️ **用纸抄**，不要存在微信、备忘录、网盘里——纸不会被黑。
纸上同时写上日期和版本号（如"v5.3 / 2026-07-05"），以后好对号。

---

## 第二步：把整个文件夹拷进 U 盘

把 `Bitcoin-Mnemonic-Offline-Generation-Tool--Beta-Version-main` 整个文件夹拖进 U 盘。

---

## 第三步：在离线设备上重新算指纹

### 离线设备是 Mac

1. 打开"**终端**"（启动台搜"终端"或"Terminal"）
2. 输入下面内容（注意 `256` 后面有一个空格），**先不要按回车**：

   ```
   shasum -a 256 
   ```

3. 把 U 盘里的 HTML 文件**直接拖进终端窗口**（路径自动填上，不用手打）
4. 按回车，屏幕显示一串 64 位字符

### 离线设备是 Windows

1. 打开"**PowerShell**"（开始菜单搜 PowerShell）
2. 输入下面内容（同样先不按回车）：

   ```
   Get-FileHash -Algorithm SHA256 
   ```

3. 把 HTML 文件**拖进 PowerShell 窗口**
4. 按回车

> Windows 显示的是大写字母（`DC80CD3E...`），大小写不影响，内容一样就行。

---

## 第四步：对照

拿出纸，对比屏幕上显示的字符串：

| 结果 | 怎么办 |
|---|---|
| ✅ 开头 8 位一致 + 结尾 8 位一致 | 文件完好，可以使用 |
| ❌ 有任何一个字符不一样 | **不要用！** 删掉文件，换个 U 盘重新拷贝，再验一次 |

---

## 一次性校验整个文件夹（进阶，可选）

文件夹里自带一份 `SHA256SUMS` 清单，在 Mac 终端里进入该文件夹后运行：

```bash
shasum -a 256 -c SHA256SUMS
```

每个文件后面都显示 `OK` 即为全部通过。

---

## 补充提醒

1. **每次使用前**，如果文件在离线设备上放了很久，重新跑一遍第三、四步再用，一分钟不到。
2. **文件一旦更新，指纹就会变。** 以后如果 HTML 文件有任何修改（升级版本等），必须重新算指纹、重新抄纸，并更新本文档中的指纹值，否则会误以为文件被篡改。
3. 本文档只解决"文件拷贝过程有没有被篡改"这一件事。正式使用前，还要完成另外两步（见 README）：
   - 用第二个独立离线钱包恢复测试助记词，比对首地址是否一致；
   - 小额收款 + 发送实测通过后，再正式使用。
